Summary: 使用markdown做本地笔记软件/编辑器

  因公司一众云笔记软件都被屏蔽用不了，只能用本地笔记，考虑到很多笔记要放图片，一开始只能用word，但那种打开速度和排版实感痛苦，所以只能是把无图片的笔记用markdown记录，有图片的用word。然而，今天才知道markdown其实是可以插入本地图片的！太棒了，那markdown作为本地笔记记录方式实在太适合了，不仅有漂亮的代码排版，字体排版，还能带目录，功能相比word效率高太多，我使用的是haroopad-markdown编辑器。
  插入本地图片方法很简单，直接用相对路径或绝对路径代替http-url即可，也有其他更666的方法，详见：Markdown插入图片–知乎
  然后发现markdown导出成html文件的功能更棒！例如我用haroopad软件把markdown文件导出成html文件会把我预览区的样式全部展示出来，唯独一点需要手工修改，那就是html的图片标签，因为默认它会把图片的源地址写成绝对路径，但一旦把图片和html文件移动到别处，那就不会显示图片了。然后我发现了个很简便的解决办法：把html文件的<img src>标签修改一下，改成相对路径，例如把图片放在跟该html文件同目录下，则直接写成<img src=pic.jpg>，在另一个目录下，则写成类似<img src=./image/pic.jpg>。然后后期传输的时候只需要把html和这个图片一块传输就行了，只要这两个文件的相对路径没被改变，图片即可正确显示。
  然后把再通过chrome浏览器等把html直接转成pdf文件，那直接连图片都不用传输了，直接传一个pdf就足够了，非常棒！
  当然，也可以直接把md转pdf，可参考教程：如何把 Markdown 文件转化为 PDF
